📱 產品概述

今心（ImXin）是一款開源情緒覺察工具，名稱取自「今心即為念」。協助使用者在情緒高漲時暫停、覺察並選擇回應方式。

🎯 產品定位

  • 以耶魯大學 Marc Brackett 的 RULER 情緒智力框架為核心方法論
  • 雙入口架構：LINE Bot 對話練習 + PWA 儀表盤 歷史回顧
  • 個人隱私優先：端側加密、本地優先、可選雲端同步
  • 支援繁體中文（台灣用語），100+ 情緒詞彙庫

👤 目標用戶

  • 希望提升情緒覺察能力的一般使用者
  • 需要親職情緒支援的父母（Parent Role）
  • 偏好低門檻、對話式互動的 LINE 用戶
  • 關注數據隱私、希望資料自主的用戶

🧠 RULER 五步框架

R Recognizing — 覺察情緒訊號（身體感受、行為線索）

U Understanding — 理解情緒成因（什麼引發了這個情緒）

L Labeling — 精確標記情緒（從 100+ 詞彙中選擇）

E Expressing — 適當表達情緒（選擇表達方式與對象）

R Regulating — 調節情緒策略（呼吸、身體掃描、認知重構）

+ NeuroCheck — 生理狀態檢視（睡眠、活動量）

前端 PWA

  • React 19.2 + TypeScript 5.9 (Strict)
  • Vite 7 + vite-plugin-pwa + Workbox
  • Capacitor 7（Android 原生包裝）
  • 狀態管理：Zustand 5 + Context + useReducer
  • 樣式：Vanilla CSS + CSS Modules（Morandi UI）
  • 開放轉換：opencc-js（繁簡）

後端 Bot Server

  • Node.js 18+ + Express 5
  • LINE SDK @line/bot-sdk 9.x
  • TypeScript + tsx 熱重載
  • 測試：Vitest 4 + supertest
  • 數據庫適配器：Memory / PostgreSQL 雙模式
  • 中間件：限速、日誌、錯誤處理

BaaS — InsForge

  • PostgreSQL + Row Level Security
  • Auth（JWT + 匿名登入）

  • Edge Functions（Deno Runtime）
  • Realtime（WebSocket Pub/Sub）
  • AI：Google Gemini REST API

🧩 功能模組詳細說明

PWA 儀表盤五大視圖 + LINE Bot 對話流程 + Agentic AI 能力

首頁 情緒覺察（Check-In Flow）

用戶進入 App 後的核心流程，基於 RULER 框架設計，提供完整版與快速版雙軌體驗。

完整 RULER 流程（~3-5 分鐘）

  • Recognizing — 情緒象限選擇MoodMeter 四象限熱力圖，用戶點選當下情緒所在的高能量/低能量 × 高愉悅/低愉悅區域。
  • Centering — 回到當下簡短引導語，協助用戶從反應模式切換到回應模式（Meta-Moment 第一步）。
  • BodyScan — 身體掃描引導覺察身體緊繃部位，將情緒與身體感受連結。
  • Labeling — 精確標記EmotionGrid 根據所選象限展示 100+ 情緒詞彙，支持多選與搜尋。
  • Understanding — 理解成因開放式問題引導用戶思考：「什麼引發了這個情緒？」
  • Expressing — 表達方式選擇表達對象與方式（獨處、書寫、對話），評估表達後的預期結果。
  • Regulating — 調節策略根據情緒類型推薦調節策略（呼吸練習、認知重構、身體活動）。
  • NeuroCheck — 生理狀態記錄睡眠時數與活動量，提供身心關聯洞察。
  • Summary — 總結與存檔完整的 RULER 日誌，可選擇繼續深入或結束。

快捷入口

  • 快速記錄（< 60 秒）：繞過完整流程，直接選擇情緒與強度，快速存檔。
  • 親職支援（Parent Role）：針對育兒情境設計的即時行動指引與情緒詞彙。
  • 繼續草稿：自動保存未完成的 RULER 流程，下次進入時提示恢復。

危機安全機制

  • 高風險情緒（絕望、無助、暴怒等）自動觸發危機介入彈窗
  • 提供台灣 24 小時心理支持熱線：安心專線 1925、生命線 1909
  • 一鍵撥號（tel: 連結），不干擾用戶主流程

首頁：MoodMeter + QuickStats + 快捷入口

歷史 時間軸（Timeline）

所有情緒覺察記錄的時間軸視圖，支持多維度篩選與搜尋。

功能特性

  • 時間軸列表：按時間倒序排列，顯示情緒、強度、象限、備註
  • 象限篩選：快速過濾紅/黃/藍/綠四象限記錄
  • 關鍵字搜尋：搜尋情緒名稱或備註內容
  • 詳情展開：點擊單條記錄查看完整 RULER 五步內容
  • 數據導出：支持 JSON / CSV 匯出（本地處理，不上傳伺服器）
  • 隱私鎖：進入歷史頁面前可要求 PIN 驗證

歷史時間軸：情緒記錄一覽

成長 儀表盤（Growth Dashboard）

以視覺化方式呈現用戶的情緒覺察習慣與趨勢。

數據視覺化

  • 情緒熱力圖：以日曆格子形式展示每日情緒象限分布
  • 趨勢曲線：情緒強度與頻率的時間序列圖
  • 象限佔比：四象限的圓餅/環形圖統計
  • 連續紀錄：當前連續天數、最長連續天數、本週/本月統計
  • 週報洞察：AI 自動生成每週情緒趨勢摘要（Edge Function）

成長儀表盤：情緒熱力圖與趨勢

成就 成就系統（Achievements）

遊戲化機制激勵持續覺察練習，所有成就條件在前端計算。

成就類型

  • 初次覺察：完成第一次 RULER 記錄
  • 連續紀錄：3 天 / 7 天 / 30 天連續記錄
  • 探索者：在四個象限都留下過記錄
  • 深度覺察：完成完整 RULER 五步流程 N 次
  • 表達練習：使用 Expressing 步驟記錄 N 次
  • 親職英雄：以 Parent Role 完成記錄（隱藏成就）

狀態追蹤

  • 已解鎖 / 未解鎖 / 新獲得（未查看）三種狀態
  • 成就解鎖時即時 toast 通知
  • 連續紀錄斷掉時溫柔提醒，無責怪語氣

成就系統：解鎖進度與獎盃展示

教練 AI 情緒教練（Agentic AI Coach）

基於 Google Gemini 的 Agentic AI 教練，具備工具調用、Session 記憶、主動行動能力。

Agentic 能力清單

  • 工具調用（Function Calling）：AI 可主動調用 4 個工具完成複雜任務
  • Session 持久化：對話記憶存儲於 PostgreSQL，跨對話保持上下文
  • 子代理路由：偵測到危機情緒時自動切換至 Meta-Moment 協議
  • 主動行動：AI 可觸發前端功能（啟動呼吸器、開啟 SOS、導航至記錄頁面）
  • 趨勢分析：查詢用戶歷史數據，提供個人化洞察與建議

4 個內建 Tools

工具名稱功能觸發場景
get_user_emotion_summary查詢用戶近期情緒摘要用戶問「我最近怎麼樣」
get_emotion_trend分析情緒趨勢與模式用戶問「我最近有沒有進步」
save_ruler_log將對話內容存為 RULER 日誌用戶完成一次口頭覺察
trigger_action觸發前端動作（呼吸/SOS/導航）AI 判斷用戶需要即時調節

Meta-Moment 危機協議

  • 當用戶輸入包含高風險關鍵詞（自傷、絕望、無意義等），自動切換為 Meta-Moment 安全模式
  • AI 不給建議，而是引導用戶暫停、呼吸、確認當下安全
  • 提供一鍵撥打 1925 / 1909 的 SOS 按鈕
  • 所有危機對話標記為高優先級，便於後續專業介入追蹤

AI 教練：Agentic 對話介面

💬 LINE Bot 對話流程

LINE 為主要入口，用戶無需安裝 App 即可開始 RULER 練習。

  • 對話式狀態機：透過 LINE Message API 維持用戶會話狀態，逐步引導 RULER 五步
  • 快捷按鈕：Quick Reply 提供情緒詞彙、強度選項、操作按鈕
  • 圖文選單：底部固定選單提供「開始覺察」、「歷史記錄」、「設定」、「緊急支援」
  • Webhook 簽名驗證：獨立 raw body 解析器確保 LINE 平台安全性
  • 數據同步：LINE 端記錄可同步至 PWA 儀表盤（需綁定帳號）

🎨 設計系統 — Luminous Morandi

受 Morandi 畫作啟發的低飽和度配色，兼顧情緒安全與視覺舒適。

情緒象限配色

每個象限對應一種情緒類型，在 MoodMeter、進度條、統計圖表中統一使用。

主題模式

  • Dark（預設）：#1a1a1a 背景，降低夜間使用眼睛疲勞
  • Light：#f5f3ef 背景，高對比閱讀場景
  • System：跟隨系統偏好自動切換

字體

  • 西文：Inter（可讀性、螢幕優化）
  • 中文：Noto Sans TC（繁體中文完整支援）
  • 8 級字階：Display (28px) → Caption (12px)

動效原則

  • 過渡時間：fast 150ms / normal 250ms / slow 400ms
  • 緩動曲線：ease-out 為主，減少焦慮感
  • 頁面切換：fade + slide，方向暗示導航層級
  • 尊重 prefers-reduced-motion

組件規範

  • 函數組件 + Hooks，禁止使用 Class 組件
  • Props 類型：React.FC<{ propName: Type }>
  • 樣式：CSS Modules（*.module.css）或 CSS 自定義屬性
  • 禁止在 JSX 中嵌入 <style> 標籤
  • 圓角系統：sm 8px / md 12px / lg 20px / xl 28px
  • 間距系統：4px base unit，s-1 (4px) → s-10 (40px)

🔐 安全機制

數據加密

  • 算法：AES-256-GCM + 隨機 IV + 鹽值
  • 密鑰：256-bit 主密鑰存儲於 IndexedDB
  • 兼容：可解密 v1 設備指紋密鑰加密的歷史數據

PIN 與密碼

  • 算法：PBKDF2-SHA256，600,000 次迭代
  • 鹽值：128-bit 隨機
  • 格式：$pbkdf2-sha256$<iterations>$<salt>$<hash>
  • 兼容：自動檢測舊版 DJB2 哈希並遷移
